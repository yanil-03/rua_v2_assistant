To add a search feature, we will integrate a web-scraping tool into your pipeline. This allows RUA to "break out" of its training data and look at the live internet.

1. Create the Research Module
We will use the ddg3 (DuckDuckGo) library because it’s fast, free, and doesn't require an API key. First, install it in your terminal:
pip install duckduckgo-search

2. Update the Orchestrator (main.py)
Add the "search for" keyword to your command interceptor.

Why this setup is better:
Hybrid Power: The script handles the internet connection (which the local LLM can't do), and then the LLM handles the "thinking" (reading the results).

No Lag: By using duckduckgo-search, you get results in milliseconds without setting up complex Google Search API keys.

Accuracy: RUA will now be able to answer questions about events that happened today, rather than just things it was trained on.
